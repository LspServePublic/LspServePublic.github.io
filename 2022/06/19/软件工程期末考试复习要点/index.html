知识要点

软件工程概述

  • 软件工程的概念定义：将系统化的、 规范的、 可度量的方法应用于软件的开发、 运行和维护过程， 即将工程化应用于软件中的方法的研究
  • 软件危机：软件危机是指落后的软件生产方式无法满足迅速增长的计算机软件需求而 导致软件开发与维护过程中出现一系列严重问题的现象
  • 软件生命周期模型（软件过程模型）
    • 瀑布模型
      1. 问题的定义及规划
      2. 需求分析
      3. 软件设计
      4. 程序编码
      5. 软件测试
      6. 运行维护
    • 增量模型
    • 螺旋模型
    • 喷泉模型

结构化分析

  • 软件需求规格说明书的内容和作用
    • 内容：
      1. 引言
      2. 任务概述
      3. 功能需求
      4. 性能需求
      5. 运行需求
      6. 非功能性需求
    • 作用：需求分析位于开发与测试的起始阶段，如果在需求阶段存在疑惑，将会导致研发出来的软件需求返工，增加软件开发成工，会导致版本不能及时交付给客户，影响深远
  • 数据流（DFD）图的画图原则
    1. 一个加工的输出数据流不应与输入数据流同名，即使它们的组成成分相同
    2. 保持数据守恒
    3. 每个加工必须既有输入数据流，又有输出数据流
    4. 所有的数据流必须以一个外部实体开始，并以一个外部实体结束
    5. 外部实体之间不应该存在数据流
  • 结构化（SA）方法的工具有哪些
    • 数据流图
    • 数据字典
    • 实体关系图
    • 系统流程图
  • 数据流图的基本图形符号
  • 软件需求分析的任务
    1. 功能需求
    2. 性能需求
    3. 可靠性和可利用性需求
    4. 出错处理需求
    5. 接口需求
    6. 约束
    7. 逆向需求
    8. 将来可能提出的要求
  • 实体关系模型（ER模型）

面向对象分析

    1. 对象模型
    2. 动态模型
    3. 功能模型

结构化设计

  • 数据流图到软件结构图的映射
  • 模块独立性
    1. 耦合：内聚是模块功能强度 (即一个模块内部各个元素彼此结合的紧密程度) 的度量。
      模块之间的连接越紧密联系越多，耦合性就越高，但模块独立性就越弱
    2. 内聚：耦合是模块之间的相对独立性 (即相互连接的紧密程度) 的度量。
      一个模块内部各个元素之间的联系越紧密，则它的内聚性就越高，相对地，它与其他模块之间的耦合性就会降低而模块独立性就越强

各类题型复习重点

  • 选择题
    复习重点中提到的相关知识的课后选择题

  • 填空题
    复习重点中提到的相关知识的课后填空题

  • 简答题

    1. 可行性研究从哪些方面研究
      1. 经济可行性
      2. 技术可行性
      3. 操作可行性
      4. 社会环境可行性至少包括两种因素：市场和政策
      5. 法律可行性
      6. 使用可行性
    2. 模块的内聚性有哪些类型、耦合性有哪些类型
    3. 数据流图转换为软件结构图的步骤
      1. 变化分析法（适用变换型系统，也就是行为类似——输入数据、变换数据、输出数据这一过程的系统）
        1. 重新分析数据流图
        2. 找出系统的逻辑输入、逻辑输出和中心变换部分
        3. 给软件结构分层
      2. 事务分析法（适用于事务型系统，也就是行为类似——根据输入的数据的特点找到合适的处理单元处理后再输出的系统）
        1. 明确事务源
        2. 规划适当的事务型结构
        3. 建立事务处理模块
        4. 定义操作模块所实现的全部细节模块
    4. 瀑布模型的适应场景：适用于传统软件工程面向过程的结构化软件开发场景
    5. 黑盒测试的概念及主要方法
      • 概念：着眼于程序外部结构、不考虑内部逻辑结构、针对软件界面和软件功能进行测试的测试方法
      • 主要方法：
        • 边界值分析法
        • 等价类划分法
    6. 白盒测试的概念及主要方法
      • 概念：全面了解程序内部结构、对所有逻辑路径进行测试的测试方法
      • 主要方法：
        • 逻辑覆盖法
          1. 语句覆盖
          2. 判定覆盖
          3. 条件覆盖
          4. 判定/条件覆盖
          5. 条件组合覆盖
        • 路径覆盖法

  • 设计题
    软件测试，对给定案例进行测试用例设计，采取逻辑覆盖法或基本路径覆盖法

    • 基本路径覆盖法中环路复杂性的计算方法有以下3种：
      • 程序的环形复杂度计算公式为 V(G)=m-n+2，其中m是程序流程图中边的数量n是结点的数量
      • 如果P是程序流图中判定结点的个数，那么V(G)=P+1
      • 如果A是程序流图中封闭区域的数目，区域的个数定义为边和结点圈定的封闭区域 数加上图形外的区域数1，那么V(G)=A+1
    • 环路复杂度也就是程序流图的基本路径数量
    • 基本路径的

    对给出的模拟系统的功能需求分析描述进行分析，采取用例图建立系统的功能模型，用类图建立系统的对象模型
